# 博主档案模板

> 每位入选博主单独建一个 `blogger-{id}.md`，按本模板填。这部分由你（人）手工总结，AI 只做格式化。
> 模板里的字段同时也是飞书表格里"博主"那张表的列定义。

## 元数据

- **id**：`xx-001` 风格，本仓库内唯一
- **昵称**：博主在小红书显示的昵称
- **小红书主页**：URL（必填）
- **粉丝量级**：写区间，例如 `5w-10w`
- **互动量级**：近 30 天平均点赞，写区间
- **认证类型**：素人 / 普通 / 专业号 / 品牌合作人

## 内容画像

### 主营赛道
1-3 个标签，例如：`轻食`、`办公室零食`、`健身餐`、`亚健康自救`。

### 选题偏好
列 5-8 条最近爆款选题或重复出现的角度，每条 1 句话。

### 表达风格
- 镜头：第一人称 vlog / 静物特写 / OOTD 混剪 / 测评对比 / ...
- 语气：知心姐姐 / 直球吐槽 / 数据党 / 文艺 / ...
- 节奏：开头钩子风格 + 是否有固定结尾

## 商单适配性

### 与 brief 的契合点
为什么这位博主适合"轻醒酸奶 0 蔗糖高蛋白希腊酸奶"，列 2-4 条。

### 风险点
有没有踩过红线的痕迹（极限词、违规话术）、是否做过竞品、限流记录。

## 推荐选题方向

为这位博主预留 2-3 条可执行的选题方向，每条一行，不写完整脚本。
脚本生成由 Coze 工作流根据 brief + 这份档案产出。

## 数据来源

列出引用过的小红书笔记 URL 和截图文件名（如有），以备评审追溯。
